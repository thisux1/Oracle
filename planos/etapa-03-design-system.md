# Etapa 03 - Design System

## Objetivo

Aplicar identidade visual base do dashboard: tokens de cor, tipografia, superficies glass, estados e utilitarios visuais compartilhados.

## Arquivos alvo

- `dashboard/src/index.css`
- `dashboard/src/components/GlassCard.jsx`
- `dashboard/src/components/StatusBadge.jsx`

## Escopo funcional

- Definir variaveis CSS da paleta Oracle.
- Definir fontes de UI e monospace.
- Criar estilos base de cards, inputs, botoes e badges.
- Estabelecer estados visuais de bot (`offline/starting/online/stopping`).

## Checklist tecnico

- [ ] CSS custom properties para bg, accent e texto definidos.
- [ ] Font stack configurada para UI e terminal/codigo.
- [ ] Classes utilitarias de glassmorphism criadas.
- [ ] Estados de foco/hover acessiveis para inputs e botoes.
- [ ] Badge de status com variantes por estado.
- [ ] Animacao de pulse para estado online.
- [ ] Contraste minimo aceitavel em textos principais e secundarios.
- [ ] Design consistente em desktop e mobile.

## Criterios de pronto

- [ ] Componentes base reaproveitaveis pelos tabs.
- [ ] Tema aplicado globalmente sem regressao visual.
